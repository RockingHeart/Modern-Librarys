# 此为开发规范

- 宏的禁用
    - 尽量避免使用宏
        - 若非必要，不可使用宏对关键字进行define

- 模板形参命名
    - 若是模板变量，或概念
        - 标准小蛇型
    - 其它情况：标准大驼峰
- 函数名及形参-变量-类类型-类型别名-概念 命名
    - 标准小蛇形

- 若函数参数的数目大于等于3个时，与若函数参数占用参数列表的空间较长时
    - 若函数体的复合语句数目大于3个时
        - 参数列表起始符号（'('）与函数名尾部隔离，隔离符号为空格，通常隔离一个，并与参数列表的第一个参数紧凑一起不换行
        - 对齐均使用空格或tab
            - 使参数类型对齐（不对齐类型属性）
            - 使参数命名对齐
        - 最后一个参数依照对齐准则，但参数列表的最后一个参数与参数列表结尾符号（')'）紧凑一起
- 函数头部属性
    - 头部属性位于函数声明之上
- 函数尾部属性
    - 若函数参数的数目大于等于3个时
        - 尾部属性位于参数列表之下
    - 若函数参数的数目小于3个时
        - 尾部属性与函数声明并行

- if consteval 块的入口标识需换行
