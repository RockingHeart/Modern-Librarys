# 往期以及当前的缺陷报告

- 现vector实现乃第三版本的重写实现，第一版本具体为设计而导致性能不达预期，第二版本同第一版本
    - 相同测试环境（有限环境）
        - 建立i循环，调用size_t构造函数，并嵌套j循环，调用push_back
            - 内外循环均为100000次
                - 测试int类型且无缓冲区，第三版本耗时8100多tick，标准库为8900多tick
            - 内外循环均为10000次
                - 测试std::string类型且无缓冲区，第三版本耗时900多tick，标准库为1700多tick
            - 若再嵌套一循环为z用于对下标z进行赋值
                - 内外循环均为1000次，而z循环为5次（对应构造参数）
                    - 测试std::string类型且无缓冲区，第三版本耗时2100多tick，标准库为2800多tick
    - 为何选择第三版本为最终实现
        - 当时判断第二版本仍然存有设计问题，故做理论思考，思考得出结果导向“box_value的设计”
        - 因剩余存储的实现为实际剩余大小，所以在push与pop等操作时，需对其计算（remain -= 1与对curent指针+=1），其计算所消耗引发速度差异（约500tick）
        - 得出理论结果之后，对其存储实现进行修改，remain为申请内存的全部大小，无需对remain进行计算
    
- 其它缺陷：float作申请内存的扩展计算会导致窄化转换与浮点计算，更改为size_t类型的扩展计算
- 视其为缺陷的原因为“Vector追求速度”
